## 🔗 배포 링크 얻기 방법

### 방법 1: GitHub + Vercel (권장)
1. **GitHub 리포지토리 생성**
   - [github.com/new](https://github.com/new) 접속
   - 리포지토리 이름: `jamaica-fitness-contract`
   - Public으로 설정

2. **코드 업로드**
   ```bash
   # 터미널에서 실행 (YOUR_USERNAME을 본인 GitHub ID로 변경)
   git remote add origin https://github.com/YOUR_USERNAME/jamaica-fitness-contract.git
   git push -u origin main
   ```

3. **Vercel 배포**
   - [vercel.com](https://vercel.com) 접속
   - GitHub으로 로그인
   - "New Project" → GitHub 리포지토리 선택
   - 자동으로 Next.js 감지되어 배포 시작

4. **환경변수 설정**
   ```env
   DATABASE_URL=postgresql://username:password@host:port/database
   ADMIN_PASSWORD=secure_password_here
   COOKIE_SECRET=random_32_char_string_here
   ```

### 방법 2: Vercel CLI (대안)
```bash
# Vercel CLI 로그인 (브라우저에서)
npx vercel login

# 배포 실행
npx vercel

# 환경변수 설정
npx vercel env add DATABASE_URL
npx vercel env add ADMIN_PASSWORD
npx vercel env add COOKIE_SECRET
```

## 🗄 데이터베이스 옵션

### 옵션 A: Vercel Postgres (가장 쉬움)
- Vercel 대시보드 → Storage → Create Database → Postgres
- 자동으로 `DATABASE_URL` 환경변수 설정됨

### 옵션 B: Supabase (무료)
- [supabase.com](https://supabase.com) 가입
- New Project 생성
- Settings → Database → Connection string 복사
- 예: `postgresql://postgres:[password]@db.[project].supabase.co:5432/postgres`
